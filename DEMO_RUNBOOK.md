# SkyAnchor 现场演示脚本

## 1. 演示目标

当前演示主线固定为：

`用户端下单 -> 调度台分配 AprilTag 0/1 -> 开始配送 -> ESP32 收到 MQTT 命令 -> 小程序跟随板端状态更新`

本版使用微信云开发聚合云函数，现场演示不要启动本地 Python 后端。

## 2. 固定演示参数

- 默认用户 ID：`receiver`
- 调度目标：`AprilTag 0` 或 `AprilTag 1`
- MQTT 设备名：`skyanchor-p4`
- MQTT topic 前缀：`skyanchor`
- 云函数：`skyanchorService`
- 云环境：`cloud1-d5g90ikff6eed3f26`

## 3. 演示前检查

### 小程序与云函数

1. 打开微信开发者工具，导入 `skyanchor-miniapp`。
2. 右键 `cloudfunctions/skyanchorService`。
3. 选择“上传并部署：云端安装依赖”。
4. 上传体验版，并确认演示手机是体验成员。
5. 进入小程序调度台，点击“自检”。

自检预期：

- 云函数：正常
- MQTT：正常
- 板端状态：可读
- 天气模式：`normal`
- 默认设备：`skyanchor-p4`

如果“板端状态”不可读，先检查 ESP32 是否已联网并连接 MQTT。

### 板端检查

串口日志至少出现：

- `wifi got ip`
- `EMQX mqtt connected`
- `system ready`

建议用第三台手机给 ESP32 开 2.4GHz 热点，用户手机和配送手机可直接使用移动数据。

## 4. 正式演示顺序

### 第一步：用户端下单

1. 首页点击“我是用户”。
2. 确认用户 ID 为 `receiver`。
3. 点击“保存”。
4. 点击“提交订单”并确认。

预期结果：

- 页面提示“订单已提交”。
- 自动进入订单详情。
- 状态为“待调度”，Tag 显示“待调度”。

### 第二步：调度台分配 Tag

1. 回到首页，点击“我是配送员”。
2. 调度台会自动刷新，也可以手动点“刷新”。
3. 在“待调度”中打开刚创建的订单。
4. 点击“选 Tag”。
5. 选择 `AprilTag 0` 或 `AprilTag 1`。

预期结果：

- 页面提示已分配对应 AprilTag。
- 主按钮变成“开始配送”。
- 订单详情事件时间线出现“分配 Tag”记录。

### 第三步：开始配送

1. 在订单详情页点击“开始配送”。
2. 小程序会等待板端 ACK。

预期结果：

- 页面提示“已开始配送”。
- ESP32 串口能看到 `start_task`。
- 事件时间线出现“开始配送”记录。
- 订单进入“识别中”，等待板端继续上报状态。

### 第四步：观察状态推进

订单详情页会自动轮询。预期状态会随板端上报推进：

- 待调度
- 识别中
- 已识别
- 执行中
- 已送达

用户端再次进入该订单，也能看到同一笔订单状态和失败原因。

## 5. 常见异常处理

### 云函数未部署

现象：页面提示“云函数尚未部署”或“云端服务未连接”。

处理：在微信开发者工具中重新上传并部署 `skyanchorService`，然后重新进入体验版。

### MQTT 不通

现象：调度台自检里 MQTT 异常，或提示“云端调度未就绪”。

处理：检查 EMQX 服务、云函数依赖是否部署完成、现场网络是否可访问公网。

### 板端状态不可读

现象：自检显示“板端状态：未读取到”，用户端不能下单。

处理：确认 ESP32 已连接 Wi-Fi 和 MQTT，并持续发布 retained state 到 `skyanchor/skyanchor-p4/state`。

### 天气管制

现象：状态条显示“天气管制”，提交订单或开始配送被拦截。

处理：这是正常保护逻辑。解除板端天气保护状态后，回到小程序点击刷新或自检。

### 未收到板端确认

现象：点击“开始配送”后提示未收到板端确认。

处理：检查 ESP32 是否订阅 `skyanchor/skyanchor-p4/cmd`，并确认会向 `skyanchor/skyanchor-p4/ack` 回 ACK。

## 6. 最稳演示节奏

时间紧时只演这条主线：

1. 调度台点“自检”，展示云端、MQTT、板端均正常。
2. 用户端提交订单。
3. 调度台打开订单并选择 `AprilTag 0`。
4. 点击“开始配送”。
5. 展示串口收到 `start_task`。
6. 展示订单详情事件时间线和状态推进。

成功标准：

- 用户可以提交订单。
- 调度端必须先分配 Tag，才能开始配送。
- 开始配送后 ESP32 收到目标号。
- 小程序能显示服务诊断、事件时间线和失败原因。
