# Embedded Competition Project

This repository contains the complete demo stack for the SkyAnchor embedded
competition project. It is intentionally kept as a mixed repository so the
ESP32-P4 firmware, CH32 controller firmware, miniapp, local server, and tooling
can be reviewed together.

## Project Layout

```text
main/                  ESP32-P4 app entrypoint and project Kconfig.
components/            ESP32-P4 firmware components.
CH32/                  CH32 lower-controller firmware and MounRiver project.
skyanchor-miniapp/     WeChat miniapp and cloud-function code.
skyanchor-server/      Local FastAPI debug backend.
tools/                 Conversion and maintenance scripts.
DEMO_RUNBOOK.md        Demo flow and troubleshooting checklist.
```

## Firmware Modules

- `components/control/`: task state, cloud command handling, CH32 link, and dock control.
- `components/camera/`: camera and video pipeline wrappers.
- `components/vision_ui/`: LVGL UI, AprilTag vision, and optional capture helpers.
- `components/drone_ai/`: ESP-DL drone classification wrapper.
- `components/bsp/`: board display and panel porting code.
- `components/app_types/`: shared firmware data types.

## CH32 Controller

`CH32/User/` contains the application logic and hardware drivers for the lower
controller. `CH32/obj/` is generated by the CH32 toolchain and should not be
tracked. MounRiver project files are kept because they are useful before demos.

## Local Runtime Files

Do not commit build outputs, generated dependencies, virtual environments,
databases, logs, model files, or private IDE/device settings. The intended local
runtime artifacts are ignored by `.gitignore`, including:

- `build/`, `managed_components/`, `ai_models/`
- `CH32/obj/`
- `.venv*/`, `.venv-ai/`, `skyanchor-server/.venv*/`
- `*.db`, `*.db-journal`, `server.*.log`
- `sdkconfig`, `sdkconfig.old`
- `project.private.config.json`

## Demo Notes

The current structure favors demo stability over a large repository reshuffle.
Avoid moving `main/`, `components/`, `CH32/`, `skyanchor-miniapp/`, or
`skyanchor-server/` before the demo unless the build and IDE paths are updated
and verified.
