# SkyAnchor 云数据库索引清单

这些索引不改变小程序功能，只用于降低云函数查询延迟和超时概率。请在云开发控制台进入对应环境的文档型数据库，为 `orders` 和 `order_events` 集合创建。

## orders

| 索引字段 | 排序 | 用途 |
| --- | --- | --- |
| `order_id` | 升序 | 订单详情、状态更新前读取单个订单 |
| `sender_id`, `created_at` | 升序, 降序 | 配送端订单列表按创建时间倒序展示 |
| `receiver_id`, `created_at` | 升序, 降序 | 用户端订单列表按创建时间倒序展示 |
| `status` | 升序 | 定时器扫描待同步订单 |

## order_events

| 索引字段 | 排序 | 用途 |
| --- | --- | --- |
| `order_id` | 升序 | 删除订单和读取订单时间线 |
| `order_id`, `created_at` | 升序, 升序 | 后续可将事件时间线排序下推给数据库 |

## 注意

- 先建普通索引即可。确认历史数据没有重复 `order_id` 后，再考虑把 `orders.order_id` 调整为唯一索引。
- 不要删除 `_id` 默认索引。
- 建索引后重新试一次：创建订单、分配 AprilTag、开始配送、送达刷新、查看订单详情、删除已结束订单。
